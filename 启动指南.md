# 项目启动指南

## 前置条件检查

在启动项目之前，请确保已安装以下软件：

### 必需软件
- ✅ JDK 1.8 或更高版本
- ✅ Maven 3.6 或更高版本  
- ✅ MySQL 8.0 或更高版本
- ✅ Docker & Docker Compose
- ✅ IDEA 2020 或更高版本（推荐）

### 验证安装

```bash
# 检查Java版本
java -version

# 检查Maven版本
mvn -version

# 检查MySQL版本
mysql --version

# 检查Docker版本
docker --version
docker-compose --version
```

## 第一步：启动基础服务

### 1.1 启动Redis和RabbitMQ

在项目根目录下执行：

```bash
# Windows PowerShell
docker-compose up -d

# 查看服务状态
docker-compose ps

# 应该看到两个服务都是 Up 状态
# NAME                   STATUS
# ecommerce-redis        Up
# ecommerce-rabbitmq     Up
```

### 1.2 验证服务是否正常

**测试Redis连接**:
```bash
docker exec -it ecommerce-redis redis-cli
127.0.0.1:6379> ping
PONG
127.0.0.1:6379> exit
```

**访问RabbitMQ管理界面**:
- 地址: http://localhost:15672
- 用户名: guest
- 密码: guest

如果能正常登录，说明RabbitMQ启动成功。

## 第二步：初始化MySQL数据库

### 2.1 连接MySQL数据库

使用MySQL客户端或Workbench连接数据库：

```bash
mysql -u root -p
# 输入你的MySQL密码
```

### 2.2 执行初始化脚本

**方法一：使用命令行**
```sql
source C:/study_cursor_dir/multi_threads/db/init.sql
```

**方法二：使用MySQL Workbench**
1. 打开MySQL Workbench
2. 连接到数据库
3. File -> Open SQL Script
4. 选择 `db/init.sql` 文件
5. 点击执行按钮（闪电图标）

### 2.3 验证数据库初始化

```sql
-- 切换到数据库
USE ecommerce_seckill;

-- 查看表
SHOW TABLES;

-- 应该看到5个表：
-- t_user
-- t_product
-- t_order
-- t_seckill_activity
-- t_seckill_order

-- 查看测试数据
SELECT * FROM t_user;
SELECT * FROM t_product;
SELECT * FROM t_seckill_activity;
```

## 第三步：配置数据库连接信息

修改各个服务的 `application.yml` 文件中的数据库密码：

需要修改的文件：
- `user-service/src/main/resources/application.yml`
- `product-service/src/main/resources/application.yml`
- `order-service/src/main/resources/application.yml`
- `seckill-service/src/main/resources/application.yml`

修改内容：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/ecommerce_seckill?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false
    username: root
    password: your_password  # 改为你的MySQL密码
```

## 第四步：使用IDEA打开项目

### 4.1 导入项目

1. 打开IntelliJ IDEA
2. File -> Open
3. 选择项目根目录 `C:\study_cursor_dir\multi_threads`
4. 点击 OK

### 4.2 等待Maven依赖下载

- IDEA会自动识别为Maven项目
- 等待右下角Maven下载依赖完成
- 可以在 View -> Tool Windows -> Maven 中查看下载进度

如果下载很慢，可以配置阿里云Maven镜像：

在 `C:\Users\Administrator\.m2\settings.xml` 中添加：
```xml
<mirrors>
    <mirror>
        <id>aliyunmaven</id>
        <mirrorOf>*</mirrorOf>
        <name>阿里云公共仓库</name>
        <url>https://maven.aliyun.com/repository/public</url>
    </mirror>
</mirrors>
```

### 4.3 检查项目结构

在IDEA左侧项目面板，应该能看到：
```
ecommerce-seckill
├── common
├── eureka-server
├── gateway
├── user-service
├── product-service
├── order-service
├── seckill-service
└── pom.xml
```

## 第五步：启动微服务

### 5.1 启动顺序

**必须按照以下顺序启动**，否则可能导致服务注册失败：

1. **EurekaServerApplication** (注册中心) - 端口8761
2. **GatewayApplication** (网关) - 端口8080
3. **UserServiceApplication** (用户服务) - 端口8081
4. **ProductServiceApplication** (商品服务) - 端口8082
5. **OrderServiceApplication** (订单服务) - 端口8083
6. **SeckillServiceApplication** (秒杀服务) - 端口8084

### 5.2 启动方法

**方法一：IDEA界面启动（推荐）**

1. 找到对应的Application类
2. 右键 -> Run 'XXXApplication'
3. 等待控制台输出启动成功信息
4. 再启动下一个服务

**方法二：使用SpringBoot Dashboard**

1. View -> Tool Windows -> Services
2. 展开 Spring Boot
3. 能看到所有Application
4. 按顺序点击绿色箭头启动

### 5.3 验证服务启动

访问Eureka注册中心：
```
http://localhost:8761
```

应该能看到以下服务已注册：
- GATEWAY (8080)
- USER-SERVICE (8081)
- PRODUCT-SERVICE (8082)
- ORDER-SERVICE (8083)
- SECKILL-SERVICE (8084)

## 常见问题解决

### Q1: 端口被占用

**错误信息**: `Port 8080 is already in use`

**解决方法**:
```bash
# Windows查看端口占用
netstat -ano | findstr 8080

# 杀死进程（记住PID）
taskkill /F /PID {进程ID}
```

### Q2: 服务无法注册到Eureka

**原因**: Eureka Server未启动或网络问题

**解决方法**:
1. 确保Eureka Server (8761) 先启动
2. 检查application.yml中的eureka地址是否正确
3. 查看服务日志是否有报错

### Q3: 数据库连接失败

**错误信息**: `Communications link failure`

**解决方法**:
1. 检查MySQL服务是否启动
2. 检查用户名密码是否正确
3. 检查数据库名称是否正确
4. 检查时区配置: `serverTimezone=Asia/Shanghai`

### Q4: Redis连接失败

**错误信息**: `Unable to connect to Redis`

**解决方法**:
```bash
# 检查Docker容器状态
docker ps | grep redis

# 重启Redis
docker-compose restart redis

# 查看Redis日志
docker logs ecommerce-redis
```

### Q5: RabbitMQ连接失败

**解决方法**:
```bash
# 检查RabbitMQ状态
docker ps | grep rabbitmq

# 重启RabbitMQ
docker-compose restart rabbitmq

# 查看日志
docker logs ecommerce-rabbitmq
```

### Q6: Maven依赖下载失败

**解决方法**:
1. 配置阿里云Maven镜像（见第四步）
2. 在IDEA中: File -> Settings -> Build -> Maven -> Importing
3. 设置 JVM options: `-Xmx1024m -DarchetypeCatalog=internal`
4. 重新导入: Maven -> Reload Project

### Q7: 内存不足

**错误信息**: `OutOfMemoryError`

**解决方法**:
1. 在IDEA中设置JVM参数
2. Run -> Edit Configurations
3. VM options: `-Xms512m -Xmx1024m`

## 停止服务

### 停止微服务

在IDEA中点击红色停止按钮，或者在控制台按 `Ctrl + C`

### 停止Docker服务

```bash
# 停止但不删除容器
docker-compose stop

# 停止并删除容器
docker-compose down

# 停止并删除容器和数据卷（慎用）
docker-compose down -v
```
