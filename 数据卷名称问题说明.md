# Docker数据卷名称问题说明

## 🔴 问题原因

当你执行 `docker volume inspect redis-data` 时报错：
```
Error response from daemon: get redis-data: no such volume
```

**原因：** docker-compose会根据**项目目录名称**自动为数据卷添加前缀。

### 实际情况

你的项目目录是：`multi_threads`

所以数据卷的实际名称是：
- ❌ `redis-data` （不存在）
- ✅ `multi_threads_redis-data` （实际名称）
- ✅ `multi_threads_rabbitmq-data` （实际名称）

## ✅ 解决方案

### 方案1：使用正确的数据卷名称（推荐，无需修改）

直接使用带前缀的名称：

```powershell
# 查看Redis数据卷
docker volume inspect multi_threads_redis-data

# 查看RabbitMQ数据卷
docker volume inspect multi_threads_rabbitmq-data

# 查看所有数据卷
docker volume ls
```

### 方案2：在docker-compose.yml中指定项目名称

如果你想使用简洁的名称（如 `redis-data`），可以在docker-compose.yml中添加项目名称：

```yaml
version: '3.8'

name: ecommerce  # 添加这一行，指定项目名称

services:
  redis:
    # ...
    volumes:
      - redis-data:/data
  # ...
```

**注意：** 如果修改了项目名称，需要：
1. 停止并删除现有容器和数据卷
2. 重新创建

```powershell
# 停止并删除（会删除数据！）
docker-compose down -v

# 重新创建
docker-compose up -d
```

⚠️ **警告：** `docker-compose down -v` 会删除所有数据卷，数据会丢失！

## 🔍 如何查看数据卷名称规则

### docker-compose的命名规则

```
数据卷名称 = {项目名称}_{数据卷名称}
```

**项目名称来源（优先级从高到低）：**
1. docker-compose.yml中的 `name` 字段
2. 项目目录名称
3. 使用 `-p` 参数指定的名称

### 示例

```yaml
# docker-compose.yml
version: '3.8'
name: myproject  # 指定项目名称

volumes:
  redis-data:
    driver: local
```

结果：数据卷名称 = `myproject_redis-data`

## 💡 最佳实践

### 1. 使用项目名称（推荐）

```yaml
version: '3.8'

name: ecommerce-seckill  # 明确指定项目名称

services:
  # ...
```

**优点：**
- 数据卷名称清晰
- 不依赖目录名称
- 便于管理

### 2. 使用环境变量

```yaml
version: '3.8'

name: ${COMPOSE_PROJECT_NAME:-ecommerce}  # 从环境变量读取，默认ecommerce

services:
  # ...
```

```powershell
# 设置环境变量
$env:COMPOSE_PROJECT_NAME="ecommerce-seckill"
docker-compose up -d
```

### 3. 使用-p参数

```powershell
# 指定项目名称
docker-compose -p ecommerce up -d
```

## 🔧 常用命令

### 查看数据卷

```powershell
# 列出所有数据卷
docker volume ls

# 查看数据卷详细信息
docker volume inspect <volume-name>

# 查看数据卷使用情况
docker system df -v
```

### 管理数据卷

```powershell
# 删除未使用的数据卷
docker volume prune

# 删除指定数据卷（需要先停止容器）
docker volume rm <volume-name>

# 备份数据卷
docker run --rm -v <volume-name>:/data -v $(pwd):/backup alpine tar czf /backup/backup.tar.gz /data
```

3. **解决方案**：
   - 使用正确的名称（`multi_threads_redis-data`）
   - 或在docker-compose.yml中指定项目名称
